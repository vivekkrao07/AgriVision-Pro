# AgriVision Pro: Technical Documentation & System Architecture

## 1. Project Overview
AgriVision Pro is a high-performance field diagnostics platform designed to detect and manage wheat diseases using AI. It provides farmers with instant, actionable insights using only a smartphone camera and a local web server.

---

## 2. System Architecture
The application follows a **Decoupled Client-Server Architecture**:

### A. Frontend (The Interface)
- **Framework**: React.js with Vite (for ultra-fast development and optimized builds).
- **Styling**: Vanilla CSS + Tailwind CSS (for layout) implementing a **Glassmorphism** design language.
- **Animations**: Framer Motion (used for layout transitions and micro-interactions).
- **Icons**: Lucide React (vector icons for sharp UI).
- **State Management**: React Context API (AuthContext) for managing user sessions and local persistence.

### B. Backend (The Intelligence)
- **Language**: Python 3.10+
- **API Framework**: FastAPI (Asynchronous, high-performance web framework).
- **Processing**: PIL (Pillow) for image manipulation and normalization.
- **Deep Learning**: PyTorch & Hugging Face Transformers.

---

## 3. The AI Model: Vision Transformer (ViT)
At the heart of AgriVision is the `wambugu71/crop_leaf_diseases_vit` model. 

### Why ViT instead of CNN?
Traditional Convolutional Neural Networks (CNNs) look at local pixel neighborhoods. **Vision Transformers** break an image into 16x16 patches and use **Self-Attention** to understand global relationships. This allows the model to identify subtle disease patterns that might be spread across the entire leaf, providing higher accuracy for complex fungal infections.

---

## 4. Data Flow Path
1. **Input**: User selects or captures an image.
2. **Pre-processing**: Frontend compresses the image to <1MB to ensure stability.
3. **API Call**: A `POST` request carries the image to the `/predict` endpoint.
4. **Feature Extraction**: The ViT model converts the image into a numerical tensor.
5. **Classification**: The model outputs a probability distribution across 4-7 disease classes.
6. **Data Enrichment**: The backend result is matched against `diseaseData.js` to provide:
   - Condition Descriptions
   - Actionable Treatment Steps
   - Severity Ratings
7. **Persistence**: The result is saved to the browser's `localStorage` for historical tracking.

---

## 5. Key Features & Logic
### AI Insight (Explainable AI)
The app uses conceptual "Visual Biomarkers" to explain AI confidence. It flags morphological deviations (structural changes in the leaf) and pattern recognition matches to build user trust.

### Dynamic Treatment Checklist
The "Maintenance Section" generates a real-time checklist. This allows farmers to track their progress in managing the disease directly from the field.

### Mobile-First Syncing
By exposing the server via `--host 0.0.0.0`, the system utilizes the local Wi-Fi network as a bridge, allowing any smartphone to act as a high-end diagnostic tool without needing cloud processing.

---

## 6. Security & Persistence
- **Authentication**: A lightweight login system that isolates data by username.
- **Local Database**: Utilizes the `localStorage` API to store JSON-serialized scan history, ensuring data survives browser refreshes without a server-side database.
